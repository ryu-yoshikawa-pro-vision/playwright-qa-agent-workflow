# 認証ルール

このファイルはサンプルです。実際の認証仕様ではありません。

## 入力ルール

- メールアドレスは必須。
- パスワードは必須。
- メールアドレスはメール形式である必要がある。

## 認証ルール

- 有効なユーザーが正しいパスワードを入力した場合、ログインに成功する。
- パスワードが誤っている場合、ログインに失敗する。
- 認証失敗時は、どの項目が誤っているかを特定できる詳細情報は表示しない。

## 遷移ルール

- ログイン成功後は `/dashboard` に遷移する。
- ログイン失敗時は `/login` に留まる。

## セキュリティ上の注意

- 認証情報をログ、証跡、成果物に残さない。
- 保存済み認証状態は Git 管理しない。
