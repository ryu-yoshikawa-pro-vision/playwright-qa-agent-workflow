# 対象プロジェクトプロファイル

このファイルはサンプルです。実在する URL、アカウント、認証情報は含みません。

## 1. 対象アプリケーション

- アプリケーション名: Mini SaaS
- 対象リポジトリ: example/mini-saas
- 対象ブランチ: main
- ローカルプロジェクトパス: ../mini-saas
- 主な利用環境: ローカル開発環境
- Base URL: `http://localhost:3000`

## 2. リポジトリとセットアップ

- 依存関係インストールコマンド: npm install
- ローカル起動コマンド: npm run dev
- 必要な環境変数: `.env.example` を参照
- セットアップ補足: サンプルのため実際の起動確認は行わない

## 3. ブラウザと Playwright CLI の利用方針

- 優先するブラウザまたはセッション: 既存の Chromium セッション
- Playwright CLI セッション名: mini-saas-local
- 既存ブラウザへの接続方法: ローカルの Playwright CLI 設定に従う
- スクリーンショット / スナップショット方針: 重要な視覚状態は snapshot と screenshot の両方で確認する
- 証跡の出力先: `artifacts/login/runs/<run-id>/evidence/`

## 4. 認証とロール

### 利用可能なロール・アカウント

- ロール: 一般ユーザー
  - アカウントまたは取得元: テスト用アカウント管理表
  - 利用可能な作業: ログイン、ダッシュボード表示確認
  - 補足: パスワードやトークンはこのファイルに記載しない

### ルール

- パスワード、トークン、Cookie、リフレッシュトークン、API キー、session storage の値をこのファイルに書かない。
- 保存済み認証状態は Git 管理しない。
- 必要なロールでアクセスできない場合、該当するブラウザ依存作業は `BLOCKED` とする。

## 5. テスト実行

- 全体テストコマンド: npm test
- 単一 spec 実行コマンド: npm test -- login.spec.ts
- デバッグ用コマンド: npm test -- login.spec.ts --debug
- レポート出力先: playwright-report/
- CI コマンド: npm run check

### ルール

- 対象プロジェクトで定義されているテスト実行コマンドを使用する。
- `playwright-cli` をプロジェクトのテストスイート実行コマンドとして扱わない。
- 対象プロジェクトのテスト実行コマンドが不明な場合、ライブテスト実行は `BLOCKED` とし、可能な範囲で静的レビューまたはブラウザ証跡レビューを継続する。

## 6. テストデータ方針

### データ種別ごとの扱い

- データ種別: ログイン用ユーザー
  - 利用可否: 利用可
  - セットアップ方法: テスト用アカウントを事前作成
  - クリーンアップ方法: 不要
  - 補足: 共有アカウントの状態変更は行わない

## 7. 安全な操作と危険な操作

### 安全に実行してよい操作

- ログイン画面の表示確認
- 入力バリデーションの確認
- ダッシュボードの表示確認

### 明示的な承認が必要な操作

- テストユーザーの設定変更
- メール送信を伴う操作
- 外部連携を呼び出す操作

### 禁止する操作

- 本番環境でのログイン試行
- 実在ユーザー情報の変更
- 認証情報を Git 管理すること

## 8. テスト生成方針

- 生成するテストの配置先: `tests/login.spec.ts`
- helper / POM の方針: まずは単一 spec に閉じる。重複が増えた場合のみ helper 化する。
- locator の優先方針: ユーザーから見える role、label、accessible name を優先する。
- assertion 方針: 遷移後の画面見出し、エラーメッセージ、ボタン状態を観察可能な形で確認する。
- cleanup / isolation 方針: テストごとに新しい page を使い、認証状態に依存しない。
