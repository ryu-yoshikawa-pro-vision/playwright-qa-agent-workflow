# 対象アプリのリポジトリ要否

このワークフローの主軸は、対象アプリのコード解析ではなく Playwright CLI による実画面探索です。

対象アプリのリポジトリは、ワークフロー開始時点の必須条件ではありません。

初回チュートリアルなどで「対象アプリのリポジトリを AI エージェントが読める場所に置く」と書かれている場合、それは Playwright Test の生成、対象プロジェクト側での実行、失敗テスト修復まで行う場合の条件です。

探索だけの場合、必要なのは対象 URL、認証方法、使用ロール、データ操作ルール、Playwright CLI の証跡です。Target Project Profile の `Local path` は `N/A` で構いません。
