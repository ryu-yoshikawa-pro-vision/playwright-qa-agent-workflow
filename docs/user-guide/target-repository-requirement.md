# 対象アプリのリポジトリ要否

このワークフローの主軸は、対象アプリのコード解析ではなく Playwright CLI による実画面探索です。

対象アプリのリポジトリは、ワークフロー開始時点の必須条件ではありません。主軸は、ブラウザで対象アプリを開き、実画面の導線、権限差分、表示状態を証跡つきで確認することです。

## 対象リポジトリなしで進められる範囲

次の作業は、対象アプリのリポジトリがなくても進められます。

- サービス全体探索
- 画面探索
- テスト計画
- テスト設計
- 生成前レビュー

必要なのは、対象 URL、認証方法、使用ロール、データ操作ルール、Playwright CLI の証跡です。

探索だけの場合、Target Project Profile の `Local path` は `N/A` または `Not used for browser exploration` で構いません。

## 対象リポジトリが必要になる範囲

次の作業では、対象アプリのリポジトリが必要になります。

- Playwright Test の生成
- 対象プロジェクト側でのテスト実行
- 失敗テスト修復

この段階では、生成先、既存 helper / fixture / POM、テスト実行コマンド、必要ならローカル起動方法も確認します。

## 既存ガイドの読み方

初回チュートリアルなどで「対象アプリのリポジトリを AI エージェントが読める場所に置く」と書かれている場合、それは Playwright Test の生成、対象プロジェクト側での実行、失敗テスト修復まで行う場合の条件として読んでください。
